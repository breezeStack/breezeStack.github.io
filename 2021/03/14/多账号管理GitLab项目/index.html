多账号管理GitLab项目

本文最后更新于：a few seconds ago

1、 先用账号登陆GitLab

2、 在项目中添加该人员，通过搜索账号

3、 生成rsa密钥

ssh-keygen -t rsa -C “15836206.zh”

/c/Users/majifu.zh/.ssh/id_rsa_15836206 //生成密钥的位置，必须将地址也打进去

id_rsa_15836206 //你的密钥名称

4、 密钥配置

Host 15836206 //使用的别名

HosName 128.64.96.200 //GitLab IP地址

UserName git //使用git命令对GitLab进行操作

IdentityFile ~/.ssh/id_rsa_15836206 //标注rsa文件公共钥匙密钥的位置

5、 将ssh密钥公钥添加到GitLab中

cat id_rsa_15836206.pub

6、 配置完ssh环境后ssh 和git clone测试

ssh -T git@Host

ssh -T git@15836206

git clone git@Host:仓库地址.git

例如：仓库地址：git@128.64.96.200:majifu.zh/pshare.git

那么git clone命令应该是:

git clone git@15836206:majifu.zh/pshare.git

7、 进行代码提交

git config user.name //查看用户名

git config user.email //查看用户邮箱

git config user.name “林江河” //配置用户邮箱

git config user.email “15836206.zh” //配置用户邮箱

正常步骤：

（1） 修改文件

（2） 拉取云端仓库到本地，进行代码冲突修复

git pull origin master

（3） 添加所有修改内容到缓存区

git add .

（4） 推送修改到本地服务器

git commit -m “备注信息

（5） 推送修改到远程服务器

git push origin master
